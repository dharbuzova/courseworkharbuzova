Кейси використання трендів

API-First розробка

Netflix прагне забезпечити користувачів персоналізованими рекомендаціями, щоб збільшити задоволення від перегляду і утримати аудиторію. Для цього Netflix використовує потужні алгоритми машинного навчання, які потребують доступу до великої кількості даних про перегляди користувачів, їх уподобання та рейтинги.

Використання API

1. Збір даних про перегляди та вподобання

Коли користувач дивиться фільм або серіал, відповідні дані передаються через API на сервери Netflix. Це включає інформацію про те, що саме переглядав користувач, коли, скільки часу він провів за переглядом і т.д.

                    POST /user/{user_id}/watch-history
                    Content-Type: application/json
                    
                    {
                    "movie_id": "12345",
                    "watched_duration": 3600,
                    "watched_at": "2024-05-20T15:30:00Z"
                    }

2. Аналіз даних та генерація рекомендацій

Зібрані дані аналізуються за допомогою алгоритмів машинного навчання, які створюють персоналізовані рекомендації для кожного користувача. На основі історії переглядів і вподобань, алгоритми обчислюють, які фільми та серіали можуть бути цікавими для конкретного користувача.

3. Відображення рекомендацій користувачеві

Коли користувач заходить на платформу, рекомендації завантажуються через API та відображаються на головній сторінці.

                    GET /user/{user_id}/recommendations
                    Content-Type: application/json

4. Адаптація інтерфейсу

API також може бути використаний для адаптації інтерфейсу користувача, наприклад, змінюючи порядок відображення категорій або додавання нових розділів, які можуть зацікавити користувача.

Netflix використовує кілька типів API для реалізації цього процесу:

  • User Data API: для збору та управління даними про користувачів.
  • Recommendation API: для надання персоналізованих рекомендацій.
  • Content API: для доступу до метаданих фільмів і серіалів.

No-Code/Low-Code розробка

Schneider Electric — світовий лідер в управлінні енергією та автоматизації, що має значний обсяг операцій у багатьох країнах.

Schneider Electric стикалася з необхідністю покращити прозорість і ефективність своїх ланцюгів поставок. Існуючі системи управління були недостатньо гнучкими для швидкого реагування на зміни ринку та вимоги клієнтів.

Використання платформи No-Code/Low-Code — Microsoft Power Apps — для розробки інструментів управління ланцюгами поставок.

Кроки реалізації:

1.Аналіз потреб:

  • Визначення ключових процесів і завдань, які потребують автоматизації та оптимізації.
  • Ідентифікація користувачів, які будуть працювати з новим додатком, і їхніх вимог.

2.Розробка додатку:

  • Використання Power Apps для створення користувацьких інтерфейсів для різних етапів управління поставками.
  • Інтеграція з існуючими системами ERP (Enterprise Resource Planning) для забезпечення безперервного потоку даних.
  • Створення дашбордів для візуалізації ключових показників ефективності (KPI).

3.Інтеграція з Power Automate:

  • Налаштування автоматизованих робочих процесів для сповіщень та оновлень статусу поставок.
  • Автоматизація процесів затвердження замовлень і рахунків.

4.Навчання та підтримка:

  • Проведення тренінгів для співробітників щодо використання нового додатка.
  • Забезпечення підтримки для користувачів у перші місяці після впровадження.

5.Оцінка та вдосконалення:

  • Регулярний моніторинг використання додатка та збирання зворотного зв’язку.
  • Внесення необхідних змін для покращення функціональності та зручності.

Результат:

  • Покращена прозорість: Завдяки новим інструментам співробітники отримали змогу в реальному часі відстежувати стан поставок і швидко реагувати на проблеми.
  • Зменшення витрат: Автоматизація процесів знизила адміністративні витрати та мінімізувала ризик помилок.
  • Підвищення ефективності: Завдяки інтеграції даних і автоматизації рутинних завдань підвищилася продуктивність команди.
  • Швидкість адаптації: Використання No-Code/Low-Code платформи дозволило швидко створювати та налаштовувати додатки під потреби бізнесу, без залучення зовнішніх розробників.

Micro frontends

Компанія "Онлайн Маркет" розвиває великий веб-додаток для електронної комерції. Додаток складається з багатьох сторінок та функцій, таких як каталог товарів, кошик, оформлення замовлення, особистий кабінет користувача тощо. З часом, код додатку ставав все більш громіздким та важким для підтримки. Різні команди розробників працювали над різними частинами додатку, що призводило до конфліктів коду та складнощів при інтеграції.

Кроки вирішення:

  • Розбиття додатку на окремі мікро-додатки.
  • Використання контейнерів для мікро-додатків.
  • Впровадження системи маршрутизації.
  • Використання API для обміну даними.

Результат:

  • Незалежна розробка та розгортання мікро-додатків.
  • Покращена масштабованість.
  • Швидші оновлення.
  • Краща стабільність.

Lazy Loading

Проблема

Etsy має велику кількість товарів, кожен з яких має зображення. При відображенні багатьох товарів на сторінці каталогу велика кількість зображень може призвести до повільного завантаження сторінки та погіршення користувацького досвіду.

Etsy використовує "ліниве завантаження" для зображень товарів на сторінках каталогу. Лише зображення, які потрібно відобразити у видимій області браузера, завантажуються автоматично. Зображення, які не переглядаються, завантажуються лише при прокручуванні сторінки користувачем.

Кроки реалізації

  • Аналіз сторінок з каталогом товарів: Визначте сторінки на вашому веб-сайті, які містять велику кількість зображень товарів, такі як сторінки каталогу, результати пошуку тощо.
  • Вибір JavaScript-бібліотеки або фреймворку: Оберіть JavaScript-бібліотеку або фреймворк, який підтримує "ліниве завантаження", такий як Intersection Observer API або Lazy Load XT.
  • Інтеграція бібліотеки в код веб-сайту: Додайте код бібліотеки до ваших HTML-сторінок та скриптів JavaScript, щоб активувати "ліниве завантаження" для зображень.
  • Встановлення атрибуту loading="lazy": Додайте атрибут loading="lazy" до HTML-тегів <img> для всіх зображень, які ви хочете завантажувати "ліниво".
  • Тестування та відлагодження: Перевірте роботу "лінивого завантаження" на різних пристроях та браузерах, щоб переконатися, що зображення завантажуються коректно і користувачам забезпечено швидкий доступ до контенту.

Результат

Після впровадження "лінивого завантаження" на веб-сайті Etsy зменшується час завантаження сторінок з каталогом товарів, що покращує користувацький досвід. Крім того, оптимізується використання пропускної здатності та ресурсів сервера, що сприяє загальній продуктивності та ефективності веб-сайту.
